This repository contains my SQL practice and assessment work completed during the **Data Technician Bootcamp**, focusing on analysing retail, sales, and operational datasets using MySQL.

## 🚀 Project Overview

Across multiple tasks, I worked with relational databases such as **Northwind** and **World DB**, writing SQL queries to:

- Retrieve and filter data  
- Sort and group records  
- Join related tables  
- Analyse retail, customer, product, and sales information  
- Produce insights for marketing, logistics, and business reporting  

This project reflects practical, industry‑aligned SQL usage.

---

## 🧠 Key SQL Skills Demonstrated

### 🔍 **[SELECT Queries](ca://s?q=Explain_SELECT_queries)**
Used to extract specific columns or full tables, such as retrieving customer lists, product inventories, and city datasets.

### 🎯 **[WHERE Filtering](ca://s?q=Explain_SQL_WHERE_clause)**
Applied conditional logic to focus on:
- High‑value products  
- Customers in specific countries  
- Cities above population thresholds  
- Orders within date ranges  

### 🔢 **[ORDER BY Sorting](ca://s?q=Explain_ORDER_BY_in_SQL)**
Sorted results alphabetically, numerically, and by date — for example:
- Most recent orders  
- Cities ranked by population  
- Products sorted by price  

### 📊 **[GROUP BY Aggregation](ca://s?q=Explain_GROUP_BY_in_SQL)**
Used to summarise data, including:
- Counting products per category  
- Calculating average populations  
- Summing quantities ordered  

### 🔗 **[JOIN Operations](ca://s?q=Explain_SQL_JOINS)**
Performed INNER, LEFT, and multi‑table JOINs to combine related data such as:
- Products with suppliers  
- Orders with customers and employees  
- Categories linked to product lists  

These JOINs enabled richer reporting and deeper insights.
